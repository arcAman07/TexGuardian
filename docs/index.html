Everything you need to ship your paper

From first draft to camera-ready, TexGuardian handles the tedious parts so you can focus on the research.

🔎

Smart Verification

Automated checks for citations, figures, tables, page limits, and custom rules defined in paper_spec.md. Catches issues before reviewers do.

⚡

LLM-Powered Fixes

Claude reads your paper, understands the issues, and generates targeted diff patches. Every fix is shown as a reviewable diff before applying.

👁

Visual Polish

Renders your PDF and sends pages to a vision model. Catches layout issues text-only analysis misses: overlapping figures, bad spacing, alignment.

💬

Natural Language

Just type in plain English. "Fix the figure overflow on line 303", "make this anonymous", "suggest more citations for related work".

🔒

Checkpoint Safety

Checkpoints before every edit. Allowlist/denylist file controls. Max changed lines. Instant rollback with /revert. Your paper is always safe.

📚

Citation Validation

Cross-references citations against CrossRef and Semantic Scholar. Finds undefined refs, unused entries, format issues, and suggests missing citations.

How it works

Three commands to go from messy draft to submission-ready paper.

1

Initialize

Run texguardian init in your paper directory. Set your venue, deadline, and custom checks.

texguardian init
2

Chat

Start the REPL. Use slash commands or plain English. The LLM reads your paper and understands context.

texguardian chat
3

Review & Apply

Every edit is a reviewable diff patch. Apply, review, or skip. Checkpoints let you roll back anytime.

[A]pply | [R]eview | [N]o

Get started

Install TexGuardian and start improving your paper.

Prerequisites

✓ Python 3.11+
✓ LaTeX (latexmk + pdflatex)
✓ AWS or OpenRouter credentials
✓ Poppler (optional, for visual polish)

# Install from source
git clone https://github.com/texguardian/texguardian.git
cd texguardian && pip install -e .

# Initialize your paper
cd /path/to/your/paper
texguardian init

# Set credentials (AWS Bedrock)
export AWS_ACCESS_KEY_ID="..."
export AWS_SECRET_ACCESS_KEY="..."

# Start chatting
texguardian chat

Frequently asked questions

Is TexGuardian free?
TexGuardian is open source (MIT license) and free to use. You only pay for LLM API calls through AWS Bedrock or OpenRouter.
Which LLM models are supported?
Claude Opus 4.5, Opus 4, Sonnet 4.5, and Sonnet 4 via AWS Bedrock. Any model from openrouter.ai via OpenRouter, including GPT-4o.
Can it break my paper?
No. Every edit is shown as a reviewable diff patch before applying. Checkpoints are created before every modification, and /revert rolls back instantly.
Which conferences are supported?
TexGuardian works with any LaTeX paper. Built-in template support for NeurIPS, ICML, ICLR, AAAI, CVPR, ECCV, ACL, EMNLP, NAACL, CHI, KDD, and more.
Do I need LaTeX installed locally?
Yes. You need latexmk and a TeX engine (pdflatex, xelatex, or lualatex) for compilation. Run texguardian doctor to check your setup.
